# /start - 会话启动初始化

**触发条件**：用户在新会话开始时调用 `/start`，或说"开始工作"、"启动"等。

## 执行步骤

在响应用户之前，**必须按顺序读取以下文件**：

```
1. requirements/INDEX.md       # 确认当前活跃 REQ
2. .claude/progress.txt        # 恢复上下文
3. 相关的 context/*/README.md  # 按需加载
```

读取完成后，输出：

```
## 会话状态

- 当前活跃 REQ: [REQ编号或"无"]
- 当前阶段: [idle/design/implementation/review/qa/ship]
- 上次工作: [简要描述]
- 下一步: [建议的下一步操作]

准备就绪，请告诉我您需要做什么。
```

## 如果当前没有活跃 REQ

询问用户：
1. 是否需要创建新 REQ？（使用 `npm run req:create -- --title "..."`）
2. 是否需要查看最近完成的 REQ？
3. 是否有其他任务需要处理？

## 禁止事项

- 禁止在没有读取上述文件的情况下直接开始实现
- 禁止跳过状态确认直接响应其他请求
